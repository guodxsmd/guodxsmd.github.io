Markdown学习记录

近来一直用markdown来写东西，标准化书写格式非常讨喜。因为没有系统性的进行过学习，经常是一边写一边问度娘语法，挺不方便。索性整理出该系语法的标准格式，一方面备查，一方面也是系统性的学习一遍。

Markdown编辑器本身是内容写作工具，本身并不支持文字排版，理论上它只是指出哪些内容是表格，哪些内容是标题，哪些是正文图片代码超链接等等。因此Markdown的语法全由一些符号所组成，这些符号经过精挑细选，其作用一目了然，其目标是实现“易写易读”。
Markdown 语法的目标是：成为一种适用于网络的书写语言。 因此，Markdown是兼容Html，HTML 是一种发布的格式，Markdown 是一种书写的格式。就这样，Markdown 的格式语法只涵盖纯文本可以涵盖的范围。不在 Markdown 涵盖范围之内的标签，都可以直接在文档里面用 HTML 撰写。不需要额外标注这是 HTML 或是 Markdown；只要直接加标签就可以了。
因此，简单来说，写一个Markdown文档，可以将直接使用Markdown语法和Html的标签混合进行使用，因为最后都会转换成Html，但要注意的是，在 HTML 区块标签间的 Markdown 格式语法将不会被处理。

段落和换行

Markdown的段落是由一个或多个连续的文本行组成，它的前后要有一个以上的空行，空行的定义是显示上看起来像是空的，便会被视为空行。比方说，若某一行只包含空格和制表符，则该行也会被视为空行。

普通段落不该用空格或制表符来缩进，我们在两个不同的文字块之间，一定要空行以示区分，不然就会被归入同一段落里。Markdown允许段落内强迫换行(插入换行符)，如果想要空一行，在插入处先按入两个以上的空格然后回车即可。

但是有时也可以使用标记来强制空行和空格，比如需要首行缩进的时候，一个空格大小的表示为：&nbsp;，空行为&emsp;，注意句末的分号;。或者在全角输入法里敲空格，那么这个空格会被认为是一个空格。

Markdown进阶使用

有时候在用Markdown时，我们不止要使用纯文本，还需要插入一些图片、链接等等，由于Markdown只是关注于纯文本，因此这些操作都要通过引用来完成。不像Word里面简单的复制粘贴。

Markdown支持两种形式的链接语法：行内式和参考式。不管是哪一种，链接文字都是用方括号[]来标记。

参考式的链接是在链接文字的括号后面再接上另一个方括号，而在第二个括号里面填入用以辨识链接的标记：
this is [an example][id] refernce-style link.

this is an example refernce-style link.
接着，在文件的任意处，把这个标记的链接内容定义出来：
[id]: http://example.com/ "这里可以写个title."

id: http://example.com/ “这里可以写个title.
链接内容的形式为：方括号(前面可以选择性地加上至多三个空格来缩进)，里面输入链接文字接着一个冒号，接着一个以上的空格或者制表符，接着链接的网址，可以选择性的接着写title内容，可以用单引号、双引号或是括弧包着，链接网址也可以用尖括号包起来，如：[id]: <http://example.com/> '这里可以写一个title'

链接辨识标签可以有字母、数字、空白和标点符号，但是并不区分大小写。
链接的定义可以放在文件中的任何一个地方，一般来说直接放在链接所在段落的后面比较好，也可以把它们放在文件最后面，就像是注解一样。
此外，用这个方法还可以将图片转化为base64编码保存在.md文件中，回头在插入图片中再进行介绍。

参考式的链接其实重点不在于它比较好写，而是它比较好读。使用Markdown的参考式链接，可以让文件更像是浏览器最后产生的结果，让你可以把些标记相关的元数据移到段落文字之外，你就可以增加链接而不让文章的阅读感觉被打断。

插入图片

很明显地，要在纯文字应用中设计一个自然的语法来插入图片是有一定难度的。Markdown使用一种和链接很相似的语法来标记图片，同样也允许两种样式：行内式和参考式。
行内式的图片语法看起来像是：


详细叙述如下：一个惊叹号!接一个方括号[]，里面放上图片的替代文字，接一个普通括号，里面放上图片的网址或本地路径（可以是相对路径或绝对路径），最后还可以用引号包住并加上选择性的title文字。
参考式的图片语法则长得像这样：
![ALT pic][id]

[id]是图片参考的名称，图片参考的定义方式则和链接参考一样：
[id]: url/to/image "title"

但是在这里有两个很大的问题：1.如果使用本地路径插入本地图片，不灵活不好分享，本地图片的路径更改或丢失都会造成Markdown文件调不出图来。2.插入网络图片，则非常依赖网络，如果是本地图片，还需要先上传到网络服务器上。
所以，如果要上传本地图片，这里也列出两个解决方法，这也是查了很多文章总结出来的，欢迎大佬们来补充指教：

1.将栖地图片上传到github
首先在github中新建一个repo，然后git clone下来，把想要的图片放到这个文件夹中上传，在github中找到这个图片的查看地址，直接在Markdown中引用就好了。

这种方法的优点在于插入式很灵活，文章上传到各个平台，图片也都基本不会丢失或找不到，但缺点就在于图片的管理很不方便，图片一旦多起来，本地仓库就会变得很大，而且文章可能涉及到很多方面，管理也不方便。不过也算是一个比较理想的解决方案了。

2.**把图片存入Markdown**文件

  • 用base64转码工具把图片转成一段字符串
  • 把字符串填到基础格式中链接的那个位置
  • 由于图片转成base64码会非常大，一般至少都要上千行，这时候会发现插入的这一长串字符串会把整个文章分割开，非常影响官文章时的体验。这时候就 可以用参考式，把大段的base64字符串放到文章末尾，然后在文章中通过一个[id]来调用，文章就不会被分割的这么乱了。

这种方法的优点就是图片真的是不会丢失，相当于直接将图片编码嵌入到文档中，但是缺点也很明显，只是一种方法，绝不推荐。

调整图片格式

利用Markdown编写文档时插入图片是默认靠左，有些时候图片设置为居中更美观，这时就需要在图片的信息前面添加如下语句：
<div align=center>![ALT pic](http://...)

设置图片大小

<img src="http:..." width = "100" height = "100" div align=right />

如上格式，在图片的最后添加width height属性设置图片大小。也可以在后面输入百分比，如width = 20% height =20%.

LaTex公式

Markdown还有一大优势就是可能支持LaTex公式。\$表示行内公式,\$$表示整行公式。更多详情可以到MathJax来了解。

这里说一个常用的，很多时候，我们在做笔记的时候，需要画箭头，然后在箭头上需要写字，这里就可以用LaTex公式来实现：

$ X\stackrel{F}{\longrightarrow}Y $

效果：$ X\stackrel{F}{\longrightarrow}Y $

LaTex支持多种箭头符号，内容很丰富，这里就不一一列举了，可以参见LaTex各种箭头符号。

内容目录

Markdown还有一个很方便的功能，就是可以根据标题自动生成目录。在段落中写上[TOC]以显示全文内容的目录结构。Markdown还有很多强大的功能，比如画流程图、序列图、甘特图、Mermaid流程图、Mermaid序列图，回头慢慢的也补充上来。

Tips:用于Markdown语法标记的符号和后面内容标记之间一定要加上至少一个空格，以便识别。在Markdown编辑中，用enter键来敲空行，最多只能空一行，如果想要在显示中有多行空行的话，就需要用HTML标签来实现了。
